# 灵宠寻踪 (Soul-Pet-Quest) 项目总结

## 1. 项目概述
“灵宠寻踪”是一款结合了 Web AR 与多模态 AI (VLM) 技术的解谜游戏全栈 MVP。项目同时包含了“灵宠解忧馆”模块，旨在通过 AI 提供趣味性的情感治愈互动。

## 2. 技术栈分析
* **前端**：原生 HTML / CSS / JavaScript
  * 实现了基于 `navigator.mediaDevices.getUserMedia` 的环境摄像头调用。
  * 实现了纯 CSS 与原生 JS 驱动的沉浸式粒子、星空背景渲染。
  * 包含了计时器、线索收集、NPC 交互弹窗等核心解谜交互逻辑。
* **后端**：Node.js + Express
  * 基于 ES Modules 构建轻量化服务端。
  * 使用 `crypto` 层等进行数据处理。
* **AI 大模型接入**：字节跳动火山引擎 (Volcengine - 豆包大模型 Doubao VLM)
  * 主要使用了 `doubao-seed-2-0-pro` 模型，结合 `ARK_API` 进行多模态请求。

## 3. 核心功能模块
### 3.1 视觉场景识别 (`/api/style-transfer`)
* 分析用户摄取的现实照片，判断当前生活场景（室内 room、街道 street、花园 garden）。
* 将现实场景动态映射至游戏内的特定治愈视觉效果。

### 3.2 灵宠解忧（多模态情感诊断） (`/api/vent-analysis`)
* 用户可以通过图片和文字进行“吐槽”。
* AI 根据设定的灵宠性格模型（“暴躁护短” 或 “温柔治愈”）对用户进行高度拟人化的情绪安抚与反击。

### 3.3 AR 解谜主玩法 (app.js)
* 涵盖了线索（Clues）收集机制。
* 与不同类型 NPC 的碰撞/点击检测。
* 全景计时器及收集进度管理。

## 4. 开发理念与交互要求 (结合用户偏好)
* **沉浸式体验**：强调一镜到底的过场动画过渡，坚决避免生硬的黑屏转场。
* **玩法一致性**：要求所有游戏机制/动作交互（例如教程引导与后续实战）维持肌肉记忆的一致性，不可出现规则反转。
* **务实的可执行性**：方案探讨需强调可执行性，拒绝纸上谈兵，新功能方案评估需结合现有架构分阶段落地。
